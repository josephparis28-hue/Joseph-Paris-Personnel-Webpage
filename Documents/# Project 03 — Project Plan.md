## 1. Project Approach

This project will follow the practical framework from *Project Management for the Unofficial Project Manager*, using a hybrid approach that balances structure with flexibility. The project will be executed in three phases:

### **Phase 1 — Planning & Documentation**

- Finalize the project scope  
- Create the project plan (this document)  
- Build the WBS and schedule  
- Conduct a TAME risk analysis  

### **Phase 2 — Website Development**

- Build the two required HTML pages  
- Apply unified styling through a single CSS file  
- Add optional JavaScript enhancements  
- Test for accessibility, responsiveness, and functionality  

### **Phase 3 — Review & Reflection**

- Conduct final testing  
- Write the retrospective  
- Finalize the README.md  
- Publish the site via GitHub Pages  
- Submit the final repository link  

This approach ensures clarity, accountability, and steady progress toward a polished final deliverable.

## 4. TAME Risk Analysis

The TAME framework evaluates **Technical, Assumptions, Market, and Execution** risks.

---

### **T — Technical Risks**

| Risk | Impact | Mitigation |

|------|--------|------------|
| CSS layout issues | Medium | Use flexbox/grid, test frequently |
| JavaScript errors | Low | Keep JS optional and modular |
| Accessibility problems | Medium | Use contrast tools, alt text, focus states |
| GitHub Pages deployment issues | Low | Test early, verify file paths |

---

### **A — Assumption Risks**

| Assumption | Risk | Mitigation |

|-----------|------|------------|
| Browser compatibility | Medium | Test on multiple devices |
| Instructor expectations | Low | Follow rubric closely |
| Time availability | Medium | Stick to schedule, avoid scope creep |

---

### **M — Market/User Risks**

| Risk | Impact | Mitigation |

|------|--------|------------|
| Website not engaging | Low | Use clean design and visuals |
| Content unclear | Medium | Keep writing concise and structured |
| Navigation confusion | Low | Use simple, consistent layout |

---

### **E — Execution Risks**

| Risk | Impact | Mitigation |

|------|--------|------------|
| Falling behind schedule | High | Break tasks into small steps |
| Over‑engineering features | Medium | Focus on required deliverables first |
| Disorganized repository | Medium | Maintain clear folder structure |
